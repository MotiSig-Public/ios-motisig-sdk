# Security policy

## Supported versions

Security updates are applied to the latest release line on the default branch (`main`). There is no long-term support policy for older major versions until versioning and release branches are explicitly documented in this file.

## Reporting a vulnerability

Please report security issues **privately** so we can assess and patch before public disclosure.

**Preferred:** use [GitHub private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability) for this repository, if enabled by the maintainers.

**Alternative:** email **security@vfcreative.com** with a clear subject line (for example, `MotiSig iOS SDK security report`). Include steps to reproduce, impact, and any suggested fix if you have one.

We aim to acknowledge reports within a few business days. Please do not open public issues for undisclosed vulnerabilities.

## Scope

This policy applies to the **motisig-sdk-ios** Swift package in this repository. It does not cover third-party apps that embed the SDK; those apps remain the responsibility of their publishers.
